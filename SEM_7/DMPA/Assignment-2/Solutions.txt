1. Answer following questions:
a. What do you mean by hypothesis testing for some parameter in a population?
Ans: Hypothesis testing is a procedure used to decide whether a claim about a population parameter (such as mean or proportion) is supported by sample data.
You set up a null hypothesis (H‚ÇÄ) representing the assumed value of the parameter and an alternative hypothesis (H‚ÇÅ) representing the competing claim.
Using sample statistics, you calculate a test statistic and determine whether the evidence is strong enough to reject H‚ÇÄ at a chosen significance level.

b. How can we use a confidence interval to conduct hypothesis testing of the population parameters?
Ans: A confidence interval gives a range of plausible values for a population parameter.
To test a hypothesis, compare the null hypothesis value with the confidence interval:
i. If the null value lies inside the interval ‚Üí we fail to reject H‚ÇÄ.
ii. If the null value lies outside the interval ‚Üí we reject H‚ÇÄ.
A 95% confidence interval corresponds to a two-tailed test at the 5% significance level.

4. Discuss about Type I and Type II errors in the context of hypothesis testing with an example.
Ans: Type I and Type II Errors
In hypothesis testing, two kinds of decision errors can occur because we base conclusions on sample data.
Type I Error: Rejecting the null hypothesis when it is actually true.
Probability of this error = Œ±.
Type II Error: Failing to reject the null hypothesis when it is actually false.
Probability of this error = Œ≤.
Example:
Let H‚ÇÄ: A machine produces items with mean weight = 50 g.
H‚ÇÅ: Mean weight ‚â† 50 g.
i. Type I Error: Concluding the machine is not producing 50 g items when it actually is.
ii. Type II Error: Concluding the machine is producing 50 g items when it actually isn‚Äôt.

6. A sample of 100 donors to a charity has a mean donation amount of $55 with a sample standard deviation of $25. Test using ùõº = 0.05 whether the population mean donation amount exceeds $50. (Refer to the attached table for the critical t-values.)
a. Define the null hypothesis and the alternative hypothesis on ùúá.

c. What is the meaning of the test statistic ùëá?
d. What are the values of the test statistic ùëáùëëùëéùë°ùëé and the p-value in this example?
e. What is our conclusion after comparing the p-value with the level of significance?
f. Interpret our conclusion so that a non-specialist could understand it.
Ans: Given: n = 100, sample mean = 55, sample standard deviation = 25, alpha = 0.05. We test whether the population mean exceeds 50.
a) Null and alternative hypotheses
H0: mu = 50
H1: mu > 50
b) Rejection rule
Reject H0 if T_data > t_critical.
From the t table: t_critical (df = 99, one-tailed, alpha = 0.05) = 1.660.
c) Meaning of the test statistic T
T measures how many standard errors the sample mean is above the hypothesized mean. A large positive T indicates evidence that the true mean is greater than 50.
d) Test statistic T_data and p-value
Standard error = 25 / sqrt(100) = 2.5
T_data = (55 - 50) / 2.5 = 2.00
From the t distribution, one-tailed p-value for T = 2.00 with 99 df is approximately 0.024.
e) Decision
p-value = 0.024 which is less than alpha = 0.05.
Therefore, we reject H0.
f) Interpretation in simple language
Based on the sample, there is strong evidence that the average donation amount in the population is greater than 50 dollars. The observed average of 55 dollars is unlikely to occur if the true average were only 50 dollars.

10. Describe the differences between the training set, test set, and validation set.
Ans: Training set: This is the data used to fit or ‚Äúlearn‚Äù the model. The model adjusts its parameters based on the patterns it finds in the training set. It is the core data on which the model is built.

Validation set: This data is not used for training. Instead, it is used to tune model choices‚Äîsuch as hyperparameters, model architecture, or stopping criteria. It helps decide which version of the model performs best before final testing. It prevents overfitting to the training set.

Test set: This is completely unseen data used only at the end to evaluate final model performance. It provides an unbiased estimate of how well the trained and tuned model will perform on new, real-world data.

11. How is the bias‚Äìvariance trade-off related to the issue of overfitting and underfitting? Is high bias associated with overfitting and underfitting, and why?
Ans: The bias‚Äìvariance trade-off explains why models underfit or overfit.
High bias means the model is too simple and makes strong assumptions.
It cannot capture the true patterns in the data.
This leads to underfitting: both training error and test error are high because the model is not flexible enough.
High variance means the model is too complex and reacts too strongly to the noise in the training data.
It fits the training data extremely well, but performs poorly on new data.
This leads to overfitting: training error is low, but test error is high.
So:
High bias ‚Üí underfitting
High variance ‚Üí overfitting
High bias is not associated with overfitting. Overfitting happens when variance is high, not bias.
The trade-off is that reducing bias usually increases variance (more complex models), and reducing variance usually increases bias (simpler models). A good model balances both.
